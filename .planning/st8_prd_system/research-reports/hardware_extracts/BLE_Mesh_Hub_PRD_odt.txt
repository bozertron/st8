Why does the world need your product?
Commercial building and factory automation represent major new market opportunities for low-power mesh networks. These markets demand industrial-grade solutions and have strict reliability, scalability and security expectations that Bluetooth mesh delivers.
Bluetooth mesh networking is a true industrial-grade solution

high security

Key Features/
Functionality
What does your product do?
The hub is a link between the mesh network and user interface on a computer.
A secure, single point access to the network
with
physical authentication
requirement which prevents unwanted access and ocntrol of the network.
The
hub acts as a provisioner/
network configurator
, adding new nodes into the network,
assigning roles to each node,
configuring their behaviour
and collecting the data from
all
the nodes
in the network
.
Network configuration data is
transferred
to the
computer/server
whenever possible,
to prevent the hub from being a single point failure,
while the hub still keeps the same information stored locally as a backup.
In case there is no
computer
/
server
connected to the hub, the hub itself could also store locally, for a period of time, all the incoming dat
a
from the nodes
(sensor data, power monitoring etc.)
(TBD)
Through the hub, user is able to
control
each individual provisioned node,
for example switch on/off or dim the light on a single node, or a group of nodes. The hub also allows the user
c
o
nfigure how
each
node interacts with other nodes in the nework,
f
or example grouping nodes and setting “scenes”. This makes it easy for entire environments, such as a room, to have each of its mesh nodes, its lights, heating system, air conditioning system and so on, all switch to a predefined, complimentary state, in a single, coordinated action.
Only one
hub
per network is required, while there may be over 1000 nodes in a network,
depending on the placement of the nodes and surroundings.
D
evices added to a network are provisioned using proven security algorithms. This uses 256-bit elliptic curves and out-of-band authentication to securely add devices.
All communication is required to be secured using AES-CCM using 128-bit keys. All mesh messages are encrypted and authenticated.

The hub

For security reasons, t
he hub itself does not have internet connectivity.
It only transfers the data to the computer/server through ethernet/serial interface.
T
he hub is based on Nordic’s nRF52840 SoC
which supports BLE and BLE Mesh protocols.
While nRF52840 has plenty of memory for firmware itself, storing provisioning data for larger networks requires additional non-volatile memory. Which is why the hub also has an external flash chip for storing network metadata.
