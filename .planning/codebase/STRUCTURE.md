- Contains: Versioned directories with `.txt` copies of earlier source files and handoff docs

- Note: Read-only archive — not loaded by the application

**`.planning/`:**

## Naming Conventions

**Files:**
- Kebab-case for all JS files: `void-engine.js`, `file-explorer.js`, `phreak-terminal.js`, `fake-stream.js`, `settings-reader.js`
- HTML files use short names: `st8.html`, `void-engine.html`

**Directories:**
- Lowercase with hyphens: `fonts/`, `snapshots/`, `docs/`
- Snapshot versions: `v{N}-{descriptor}` pattern (e.g., `v3-tap8-keyboard`)

**CSS Classes:**
- Component-prefixed: `.explorer-*`, `.phreak-*`, `.void-*`, `.sirkit-*`, `.panel-*`
- BEM-like but not strict: `.phreak-line--input`, `.phreak-line--stdout`, `.crumb-btn--last`

**JavaScript:**
- Panel state objects: camelCase module-scoped (`explorerState`, `phreakState`)
- Private functions: underscore prefix (`_renderExplorer`, `_fetchViaWebSocket`, `_simulateCommand`)
- Public API: attached to `window.*` with PascalCase names (`window.VoidFileExplorer`, `window.PhreakTerminal`)
- Constants: UPPER_SNAKE_CASE (`BODY_FONT`, `VIRTUAL_SCROLL_THRESHOLD`, `MEDIA_COMMANDS`)

## Where to Add New Code

**New Panel (e.g., music player, image viewer):**
1. Create `new-panel.js` in project root (follow `file-explorer.js` pattern)
2. Expose `window.VoidNewPanel = { mount, ... }` at bottom
3. Add overlay HTML in `st8.html` (copy existing panel overlay structure)
4. Add button to `.dock-row` in `st8.html`
5. Register in the `panels` object in the inline controller (`st8.html:717-793`)
6. Add CSS styles in the `<style>` block with `.newpanel-*` prefix

**New void-engine Feature:**
- Obstacle types: Add to `void-engine.js` — extend `layoutColumn()` blocked-interval logic
- Cursor behavior: Modify cursor positioning block at `void-engine.js:324-333`
- Input handling: Extend `keydown` handler at `void-engine.js:154-172`

**New Phreak Command:**
- Add to `_simulateCommand()` map in `phreak-terminal.js:103-123` for offline fallback
- Add to `MEDIA_COMMANDS` in `phreak-terminal.js:140-145` if it's a media command
- Or handle directly in `phreakExecute()` for custom logic

**New Settings Category:**
- Add defaults in `settings-reader.js:17-25` (follow `DEFAULT_VOIDFLOW` pattern)
- Add getter in `SettingsReader` class (line 65-68)
- Access via `st8Settings.newCategory.key`

**Shared Utility:**
- Add to an existing file if closely related (e.g., geometry helpers in `void-engine.js`)
- Or create a new `.js` file in root and load via `<script>` tag in `st8.html`

## Script Loading Order

The load order in `st8.html` matters because panels depend on globals:

```
1. void-engine.js    (type="module" — independent, async)
2. window.escapeHtml (inline <script> — defines global helper)
3. file-explorer.js  (<script> — attaches window.VoidFileExplorer)
4. phreak-terminal.js (<script> — attaches window.PhreakTerminal)
5. Inline controller (<script> — uses all of the above)
```

**Critical:** `window.escapeHtml` must be defined before `file-explorer.js` loads, because `file-explorer.js` references it in template strings. The panel controller must run last because it calls `mount()` on both panels.
